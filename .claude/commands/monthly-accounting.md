# monthly-accounting

月次の経理を確定し、個人用ダッシュボードの財務パネルに反映するスキル。
毎月初（前月が終わったタイミング）に実行する。

`/check-payment` が「金額の確認」担当なのに対し、こちらは
**確認した金額を経理に確定 → ダッシュボードに反映 → push** まで行う。

## 引数
- 対象月（例: 「5月分」）。未指定なら直近の終わった月を対象にする。

## 手順

### 1. 金額を確定する
- `/check-payment` のロジックで各クライアントの最新PDFを読み、対象月の金額を確定する
  - and°：支払い明細書の合計支払額（確定値。明細が未発行なら**見込み**として扱う）
  - ライズファン・かつや：発行請求書の金額
- 明細未発行で確定できない場合は、ユーザーに見込み額を確認する（推測で埋めない）
- ⚠️ 確証のない金額は入力しない。見込みは必ず「見込」と明記する

### 2. ダッシュボードの財務パネルを更新する
対象ファイル: `/Users/furusawasuisei/Documents/Claude Code/dashboard.html`

- 月次売上テーブルに対象月の行を追加／更新（確定 or 見込みを区別）
- 「年間累計（見込み含む）」KPIと注記を再計算して書き換える
- 変動報酬の内訳（編集者管理費・インセンティブ・売却報酬等）は注記に残す

### 3. 財務索引を更新する
- `02_finance/index.md` の各クライアント報酬欄に変更があれば差分を提示し、
  確認を取ってから書き込む（⚠️ 財務ファイルは確認なしに書き換えない）

### 4. 結果を表で報告する

```
## {対象月} 経理確定サマリー

| クライアント | 種別 | 金額 | 確定/見込 |
|:-----------|:-----|-----:|:--------|
| and°       | 入金 |      |          |
| ライズファン | 請求 |      |          |
| かつや      | 請求 |      |          |
| 合計        |      |      |          |

年間累計: ¥XXX（確定 ¥XXX ＋ 見込み ¥XXX）
法人化トリガー該当: なし / あり（該当条件）
```

- 法人化トリガー（月75万超2ヶ月連続・年800万が見えた等）に該当したら必ず指摘する → [[project_incorporation]]

### 5. ダッシュボードをpushする
財務パネル更新後：

```bash
git add dashboard.html
git commit -m "Monthly accounting update ({対象月})"
git push
```

## 注意事項
- PDFが存在しない・読めない場合は「ファイルなし」と報告し、見込みで進めるか確認する
- and°は月末発行の支払明細が確定値。明細が出るまでは見込みのまま据え置く
